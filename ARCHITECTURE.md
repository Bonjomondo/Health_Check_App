## 蓝牙通信协议

### 连接方式
- **协议**: 经典蓝牙 (Bluetooth Classic)
- **配置文件**: 串口配置文件 (Serial Port Profile - SPP)
- **UUID**: `00001101-0000-1000-8000-00805F9B34FB`
- **数据格式**: JSON字符串，以换行符分隔

### 数据接收 (MCU → App)

#### 传感器数据
```json
{
  "heartRate": 75,
  "bloodOxygen": 98,
  "bodyTemperature": 36.5,
  "environmentTemperature": 25.0,
  "humidity": 60,
  "motionStatus": "SEDENTARY",
  "steps": 1234,
  "battery": 85,
  "timestamp": 1637500000000
}
```

#### 设备状态
```json
{
  "battery": 85,
  "charging": false,
  "timestamp": 1637500000000
}
```

### 命令发送 (App → MCU)

#### 控制指令
```json
{
  "command": "START_MEASURE",
  "timestamp": 1637500000000
}
```

支持的命令：
- `START_MEASURE` - 开始测量
- `ALARM_FALL` - 跌倒报警
- `ALARM_FEVER` - 高烧报警
- `ALARM_HEART_RATE` - 心率异常报警

## 数据流图

### 正常监测流程

```
1. 用户在设置页面扫描并连接蓝牙设备
   │
   ▼
2. App建立蓝牙SPP连接
   │
   ▼
3. 用户点击"开始测量"按钮
   │
   ▼
4. App通过蓝牙发送 START_MEASURE 命令
   │
   ▼
5. MCU接收命令，启动MAX30102采集
   │
   ▼
6. MCU采集数据并通过蓝牙发送JSON数据
   │
   ▼
7. App接收数据并更新UI (每500ms-1s)
   │
   ▼
8. App检查阈值
   │
   ├─ 正常 → 显示绿色状态
   │
   └─ 异常 → 弹窗 + 震动 + 发送报警命令给MCU
              │
              ▼
           MCU触发蜂鸣器
```

### 跌倒检测流程

```
1. MPU6050检测到加速度异常
   │
   ▼
2. MCU判定为跌倒，通过蓝牙发送状态
   │
   ▼
3. App接收到 motionStatus: "FALL_DETECTED"
   │
   ▼
4. App弹出警报对话框
   │
   ▼
5. App震动提醒
   │
   ▼
6. App通过蓝牙发送 ALARM_FALL 命令
   │
   ▼
7. MCU触发蜂鸣器报警
```

## 技术栈

### Android App
- **语言**: Java
- **最低SDK**: Android 7.0 (API 24)
- **目标SDK**: Android 14 (API 36)
- **UI框架**: Material Design Components
- **图表库**: MPAndroidChart 3.1.0
- **通信**: Android Bluetooth Classic (SPP)
- **架构模式**: MVC

### 单片机系统
- **传感器**:
  - MAX30102: I2C, 心率/血氧
  - DS18B20: 1-Wire, 体温
  - DHT11/DHT22: 单总线, 温湿度
  - MPU6050: I2C, 加速度/陀螺仪
- **通信**: 蓝牙模块 HC-05/HC-06
- **报警**: 蜂鸣器
- **电源**: 锂电池 + 充电管理

## 蓝牙配置

### 单片机端配置
```
蓝牙模块: HC-05 或 HC-06
波特率: 9600 或 115200
数据位: 8
停止位: 1
校验位: None
流控制: None
```

### 数据传输格式
- 每条JSON消息以换行符 `\n` 结尾
- 使用UTF-8编码
- 建议数据发送频率: 1Hz (每秒一次)

## 性能优化

1. **UI刷新限流**: 500ms-1s刷新一次，避免频繁更新导致卡顿
2. **数据缓存**: SharedPreferences缓存设置，避免频繁读写
3. **异步处理**: 蓝牙通信在工作线程，UI更新切换到主线程
4. **自动重连**: 支持保存设备地址，应用启动时自动重连
5. **连接管理**: 单一连接模式，避免多重连接冲突

## 安全考虑

1. **配对认证**: 使用蓝牙配对机制确保连接安全
2. **权限管理**: 运行时请求蓝牙和位置权限
3. **数据验证**: JSON解析异常处理，防止崩溃
4. **连接验证**: 检查设备是否已配对后再连接
